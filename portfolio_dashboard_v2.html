운영 원칙
Public repo에는 실 portfolio.json을 올리지 않습니다.
Viewer는 UI 전용 — 데이터 소스가 될 수 없습니다.
데이터 입력은 파일 업로드 / 붙여넣기 / localStorage만 허용합니다.
fetch('./portfolio.json') 구조는 사용하지 않습니다.
추가 자금 중심 리밸런싱을 기본 원칙으로 합니다.
❌ 매도 기반 리밸런싱 → ✔ 추가 자금 기반 리밸런싱
Telegram 자동 발송은 private advisor repo에서만 수행합니다.
브라우저 데이터는 Actions에서 접근 불가 — 반드시 private JSON 기준.
API 키 / 토큰은 절대 HTML에 포함하지 않습니다.
GitHub Actions secrets로만 관리합니다.

portfolio.json 붙여넣기
실제 portfolio.json을 public repo에 올리지 말고, 여기 붙여넣거나 파일 업로드로만 사용하세요.
이 창을 닫으면 입력 내용은 사라집니다. 브라우저 저장 버튼을 눌러야 유지됩니다.
